## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🔗 **实时监控** | 轮询 TRON 区块链，监控高度尾数为 `00/20/40/60/80` 的区块 |
| 🧠 **61 条预测规则** | 跟随/反向、长龙追踪、马尔可夫链、哈希分析、Hurst 指数、集成投票等 |
| 📊 **自适应权重** | 根据规则历史胜率和近期表现自动调整权重 |
| 🔔 **钉钉推送** | 结果推送 + 下期预测推送，含规则胜率统计 |
| 🖥 **Web 前端** | 实时数据显示，规则开关/反转，配置管理 |
| ⚡ **熔断保护** | 连败时自动跳过预测，防止追单 |
| 🔄 **逆向模式** | 连败时自动反转预测方向 |
| 📈 **回测系统** | 内置回测引擎，可评估规则组合表现 |

## 🧠 规则引擎

系统包含 **61 条预测规则**，分为以下几类：

| 类别 | 规则 | 说明 |
|------|------|------|
| **基础规则** | R1-R9 | 最近 N 球跟随/反向、多数跟随 |
| **长龙策略** | DRAGON | 短龙追、中龙跟、大长龙砍 |
| **统计规则** | R10-R20 | 动量分析、统计显著性、马尔可夫链 |
| **技术规则** | R21-R40 | 哈希分析、RSI、布林带、Hurst 指数、贝叶斯 |
| **高级规则** | R41-R50 | 集成投票、多周期共振、熵分析、模式匹配 |
| **尖端规则** | R51-R61 | 条件熵、马尔可夫切换、蒙特卡洛、游程检验 |

每条规则都有**自适应权重**，根据历史胜率和近期表现自动调整。

## 🚀 快速开始

### 前提条件

- Node.js 18+
- 一个钉钉机器人 Webhook（可选）

### 安装

```bash
# 克隆仓库
git clone https://github.com/ly5201314gjx/tron-predictor.git
cd tron-predictor

# 安装依赖
npm install

# 启动服务
node server.js
```

### 配置

编辑 `data/config.json`：

```json
{
  "dingWebhook": "https://oapi.dingtalk.com/robot/send?access_token=your_token",
  "dingModeHighWinRate": true,
  "dingModeHighWinThreshold": 55,
  "pollingInterval": 12000
}
```

### 访问

- Web 界面：`http://localhost:3456`
- API 状态：`http://localhost:3456/api/status`

## ⚙️ 核心算法

### 置信度计算

不再依赖单双票数差距，而是基于**规则质量**：

```
置信度 = 一致性(25%) + 历史胜率(35%) + 近期表现(40%)
```

- **一致性**: 支持最终预测的规则占比
- **历史胜率**: 支持方规则的平均历史胜率
- **近期表现**: 支持方规则近 30 次平均胜率

### 自适应权重

每条规则权重根据历史表现动态调整：

```
有效权重 = 基础权重 × 自适应因子 × 近期因子
自适应因子 = clamp(0.8 + (胜率 - 0.5) × 1.2, 0.4, 1.3)
近期因子 = clamp(0.85 + (近30次胜率 - 0.5) × 0.6, 0.5, 1.2)
```
