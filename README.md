## 🌟 这是什么？

**TRON Predictor** 是一套基于 TRON 区块链哈希的奇偶（单/双）预测系统。它通过分析链上区块哈希的最后一位数字（字母向前搜索），结合 61 条数学与统计规则，以加权投票的方式产出预测，并通过钉钉机器人实时推送结果。

**核心能力一览：**

| 🚀 零延迟 | 🧠 61 条规则 | ⚡ 智能熔断 | ⏱ 实时倒计时 |
|:---:|:---:|:---:|:---:|
| WebSocket 实时数据源 | 加权投票 + 自适应权重 | 连败自动暂停 | SVG 圆环 + 秒级走动 |
| API 轮询备用 | 三维置信度算法 | 递增冷却保护 | 后端驱动精准同步 |

## ⚡ 核心特性

### 🔌 零延迟 WebSocket 数据源

传统轮询需要 3-10 秒等待 API 响应，而我们通过 WebSocket 实现了**零延迟**：

```
传统模式:  API 请求 → 等待 3-10s → 收到数据 → 处理 → 推送
零延迟:    WebSocket 推送 → 即时处理 → 即时推送
```

- **v2hs9.com WebSocket** — `wss://www.v2hs9.com/ws/` 实时接收 TRON 区块
- **双源架构** — WebSocket 为主、API 轮询为备，自动切换
- **CF Worker 代理** — `tron-api.lg0304.xyz` 全球边缘节点加速 API 调用，延迟降低 70%

### 🧠 61 条预测规则引擎

每条规则都是一个独立的数学模型，通过加权投票产出最终预测：

| 类别 | 规则编号 | 典型算法 |
|------|----------|----------|
| **基础跟随/反向** | R1-R9 | 最近 N 球多数跟随、反向多数、振荡识别 |
| **动能/动量** | R10-R17 | 指数加权动能、线性回归斜率、数值均值偏移 |
| **马尔可夫链** | R41-R42 | 2 阶/3 阶状态转移概率 |
| **哈希分析** | R25, R39, R43 | 尾字节关联、首尾字节关联、哈希连续重复 |
| **统计检验** | R60, R61 | 游程分布卡方检验、多指标一致性 |
| **集成投票** | R49, R50 | 自适应集成投票、趋势一致性强化 |
| **龙策略** | DRAGON | 短龙追（2-3）、中龙跟（4-6）、大长龙砍（≥7）|

**加权投票流程：**

```
规则 R1  → 预测: 双, 置信: 60%, 权重: 1.00  ─┐
规则 R3  → 预测: 双, 置信: 67%, 权重: 1.00  ─┤
规则 R6  → 预测: 单, 置信: 61%, 权重: 0.90  ─┤ → 加权求和
规则 R11 → 预测: 双, 置信: 92%, 权重: 1.00  ─┤   单总分 vs 双总分
规则 R24 → 预测: 双, 置信: 81%, 权重: 1.10  ─┤   → 高者胜出
... (共 35 条触发)                              ─┘
                                          最终: 双 (67.6%)
```

### 🎯 置信度算法 2.0

不再看"谁的票多"，而是看"投票的质量"：

```
综合置信度 = 一致性 × 25% + 历史胜率 × 35% + 近期表现 × 40%

一致性   = 支持最终预测的规则占比（如 35 条中 28 条投双 → 80%）
历史胜率 = 支持方规则的平均历史胜率（如 28 条平均 63.2%）
近期表现 = 支持方规则近 30 次平均胜率（如 28 条近期 65.1%）
```

**输出范围：** 50% ~ 92%，更真实反映预测可信度。

### ⚡ 智能熔断系统

让系统学会"认输"——在坏趋势中按下暂停键：

| 参数 | 值 | 说明 |
|------|-----|------|
| 连败阈值 | 2 次 | 连输 2 把自动暂停 |
| 基础冷却 | 2 个区块 | 给系统喘息时间 |
| 递增系数 | +1/次 | 第 1 次跳 1 块，第 2 次跳 2 块... |
| 最大冷却 | 5 个区块 | 硬上限，防止过度等待 |
| 重置条件 | 预测正确 | 一旦赢了，冷却计数归零 |

**回测成果：** 最高连败从 11 次降至 2 次，胜率提升 0.6%。

精确到秒的 50 秒周期倒计时，后端驱动，前端同步：

- 🔧 **后端权威** — `countdownValue` 每秒 -1，新区块到达时重置为 50

### 📊 三种推送模式

| 模式 | 触发条件 | 推送内容 |
|------|----------|----------|
| **连胜连败** | 连续相同结果 ≥ 阈值 | 结果 + 下期预测 + 智能提示 |
| **输赢交替** | W/L 交替 ≥ 阈值 | 交替轮次 + 预测 + 跟买/反买提示 |
| **AA 结构** | 连续连赢/连败 ≥ 阈值 | AA 轮次 + 趋势方向 + 策略建议 |

### 🔄 周期检测引擎

实时感知链上数据的分布节奏：

- 📊 **滑动窗口** — 近 50 球单双分布实时分析
- ⚖️ **偏移检测** — 分布偏移超过 8% 时触发
- 🔄 **智能加权** — 偏单周期加权单，偏双周期加权双
- 📈 **偏移显示** — 前端实时展示 `偏单 4.0%` 或 `偏双 2.1%`

## 📊 单双判定规则

游戏基于 TRON 区块哈希的**最后一位数字**：

```
哈希示例: a3f8e2b1c9d04765...
最后一位: 5 → 奇数 → 单

哈希示例: a3f8e2b1c9d0476e...
最后一位: e → 字母 → 向前搜索 → 6 → 偶数 → 双
```

**字母后退逻辑：**
| 最后一位 | 处理方式 | 结果 |
|----------|----------|------|
| `0-9` | 直接使用 | 奇→单，偶→双 |
| `a-f` | 向前搜索最近的数字位 | 找到数字后判断奇偶 |

## ⚙️ 核心算法详解

### 自适应权重

每条规则的权重根据历史表现动态调整：

```
有效权重 = 基础权重 × 自适应因子 × 近期因子

自适应因子 = clamp(0.8 + (历史胜率 - 0.5) × 1.2, 0.4, 1.3)
近期因子   = clamp(0.85 + (近30次胜率 - 0.5) × 0.6, 0.5, 1.2)
```

- 胜率 > 50% 的规则获得更高权重（最多 ×1.3）
- 胜率 < 50% 的规则被降权（最少 ×0.4）
- 近期表现对权重影响更大（40% 权重占比）

### 周期检测

```
偏移 = (近N球单数 - 近N球双数) / N

|偏移| > 阈值 → 加权调整预测方向
偏移为正 → 偏单周期 → 加权单
偏移为负 → 偏双周期 → 加权双
```

### 龙策略

```
龙长度 2-3  → 短龙追   → 给龙方向 +0.25~0.35 分
龙长度 4-6  → 中龙跟   → 给龙方向 +0.10~0.20 分
龙长度 ≥7   → 大长龙砍 → 给反方向 +0.30~0.55 分
```
