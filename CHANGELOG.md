# 更新日志 (Changelog)

## v1.1.0 - 2026-06-23

### 🔧 推送逻辑修复（核心）

- **修复钉钉重复推送**：每个区块现在严格只推 2 条消息（1 结果 + 1 预测），不再多推
- **修复 `fetchOnce` 并发**：不再强制重置 `isProcessing`，避免与轮询并发导致重复
- **修复目标高度计算**：`ceil((height+1)/20)*20` 确保预测永远指向下一块，不会重复推当前块
- **`addBall` 提前到推送前**：先入库再推送，防止并发 handleBlock 通过去重检查
- **删除备份定时器**：移除 `scheduleNextPrediction()` 死循环风险，由 `processNewBlocks` 统一推一次
- **推送顺序固定**：`await pushResult()` 等结果推完再推预测，钉钉消息顺序正确

### 🧠 置信度算法升级

- **新算法**：不再看单双票数差距，改为基于规则质量
- **三大维度**：一致性(25%) + 历史胜率(35%) + 近期表现(40%)
- **映射范围**：50% ~ 92%，更真实反映预测可信度

### 📊 推送内容优化

- **紧凑规则显示**：`R1(60%) R2(55%) R3(72%)` 一行展示，按胜率排序
- **近 100 球统计**：显示近期胜率和单双分布
- **平均胜率**：预测和结果推送均显示参与规则的平均胜率

### 🛡 安全性提升

- **`highestPushedHeight` 持久化**：保存到 state.json，重启后不重复推送旧结果
- **fetcher 15 秒超时**：AbortController 防止 API 请求无限挂起
- **定时器空闲检测**：区块停更 3 分钟后暂停备份定时器

### ⚙️ 配置调整

- **推送阈值降低**：`dingModeHighWinThreshold` 从 62% 调至 55%
- **前端同步**：默认值、阈值、紧凑格式均与后端一致
